1. Общее описание проекта AURORA
Цель: Создать масштабируемую, безопасную и высокопроизводительную панель управления протоколами VPN на базе ядра Xray-core (и аналогов), поддерживающую мульти-протокольность, управление пользователями и мониторинг ресурсов.
Конкурентные ориентиры: Marzban, Remnawave, 3X-UI.
2. Технический стек
При генерации кода следуй этому стеку:
Backend: Python (FastAPI) или Go (Gin/Fiber).
Frontend: React.js или Vue.js (TypeScript, Tailwind CSS, Shadcn UI).
Core: Xray-core (gRPC API для взаимодействия).
Database: PostgreSQL (основные данные), Redis (кэширование и статистика в реальном времени).
Очереди задач: Celery или аналоги для обработки лимитов и уведомлений.
3. Поддерживаемые протоколы и транспорт
Панель должна уметь конфигурировать:
Протоколы: VLESS, VMess, Trojan, Shadowsocks, Shadowsocks-2022, Hysteria2, TUIC v5.
Транспорт: TCP, HTTP, WebSocket, gRPC, QUIC.
Безопасность: TLS, Reality, XTLS-Vision.
4. Архитектурные требования
Модуль управления (Core Manager)
Динамическое создание и изменение config.json для Xray.
Взаимодействие с ядром через gRPC без перезагрузки процесса (hot reload конфигураций).
Управление пользователями
Система подписок (Subscription API).
Лимиты по трафику (GB), по дате истечения и по количеству одновременных IP-подключений.
Генерация ссылок и QR-кодов (v2ray/clash/sing-box форматы).
Мониторинг и аналитика
Сбор статистики трафика в реальном времени.
Логирование подключений.
Dashboard с графиками нагрузки на CPU/RAM и сетевой активности.
5. Пошаговый план реализации (Инструкции для DeepSeek)
Этап 1: Проектирование Базы Данных
Запрос к модели: "Разработай схему базы данных PostgreSQL для VPN панели. Учти таблицы для пользователей, узлов (nodes), прокси-инbound'ов, лимитов трафика и логов сессий. Напиши модели на SQLAlchemy (или GORM)."
Этап 2: Интеграция с Xray-core (gRPC API)
Запрос к модели: "Напиши сервис на Python/Go для взаимодействия с Xray-core через gRPC. Реализуй функции: добавление пользователя в существующий inbound, удаление пользователя, получение статистики трафика конкретного пользователя (Uplink/Downlink)."
Этап 3: Система подписок (Subscription Manager)
Запрос к модели: "Реализуй API эндпоинт, который выдает конфигурацию в формате Base64 или YAML (для Clash/Sing-box) в зависимости от User-Agent запроса. Интегрируй систему проверки валидности токена подписки."
Этап 4: Веб-интерфейс (Frontend)
Запрос к модели: "Создай современный дашборд администратора на React. Ипользуй TanStack Query для данных и Recharts для визуализации трафика. Реализуй форму создания нового пользователя с выбором протоколов и лимитов."
Этап 5: Мульти-нодность (Distributed Nodes)
Запрос к модели: "Опиши архитектуру подключения удаленных серверов (узлов) к центральной панели. Реализуй метод синхронизации пользователей между центральным сервером и удаленным Xray-core через защищенный канал."
6. Требования к безопасности
JWT-авторизация для API.
Шифрование чувствительных данных в БД.
Автоматическая очистка устаревших логов.
Поддержка Fail2Ban и защита от перебора паролей.
7. Формат вывода кода
При написании кода следуй принципам:
Clean Architecture: Разделение на слои (Models, Services, Controllers).
Docker-ready: Каждый компонент должен иметь Dockerfile и общий docker-compose.yml.
Документация: Swagger/OpenAPI для всех эндпоинтов.
